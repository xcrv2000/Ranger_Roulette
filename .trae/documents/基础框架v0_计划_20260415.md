## 方案概述（高层）

采用“主流程 + UI 路由 + 地图节点状态”三件套闭环：
- `GameFlow`：顶层状态机（封面 → 主界面/地图 → 节点内容 → 结算/胜利 → 返回封面）。
- `MapController`：管理线性固定节点序列、当前位置、进入/完成节点与推进逻辑。
- `UIController`：实例化并切换 UI（封面 UI、主界面 UI、胜利 UI），将按钮事件转发给 `GameFlow`。

UI 落地策略：
- 新建封面场景 `TitleScreen`，按封面草图布置：上方标题，中部按钮区（New/Continue），底部跑道/角色占位。
- 重构 [ui_root.tscn](file:///e:/2025+/AI编程/漫游轮盘/scenes/ui/ui_root.tscn) 为“主界面 UI”：顶部数据/金币条、中部 map 区域、战斗舞台占位区、底部 relic/slot 区域；并提供一个“节点内容面板”用于展示战斗占位或空白事件占位。
- 新建胜利结算场景 `EndScreen`：展示“胜利/游戏结束（占位）”与“返回封面”按钮。

## 固定节点序列（线性）

为满足“可推进并最终结束游戏”，本阶段采用固定序列（可在脚本里集中定义，便于未来替换为 MapGenerator）：
1. 战斗（普通，占位）
2. 事件（占位）
3. 商店（占位）
4. 休息处（占位）
5. 战斗（Boss，占位）
6. 胜利结算（EndScreen）

地图“在节点之间移动”的最低实现：
- Map 区域显示节点条（如 1..5）。
- 提供左右按钮或可点击节点以切换 `selected_index`（限定在 `0..current_index` 或 `0..current_index+1` 之间）。
- “进入节点”只允许进入 `selected_index == current_index` 的节点，避免越级。

## 验证步骤（实现后执行）

在 Godot 编辑器内运行项目并验证：
1. 启动 → 进入封面：New 可点，Continue 不可点且有占位提示。
2. 点击 New → 进入主界面：顶部 data/$、中部 map、底部 relic/slot 可见。

6. 推进到 Boss 战并胜利 → 进入胜利结算界面；点击返回封面可重新开始。
