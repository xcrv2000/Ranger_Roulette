# 战斗控制器支持漫游枪手卡牌（计划）

## Summary

- 在现有“轮盘=弹匣”的基础上，实现【消耗】、随机打出子弹、游走、多段伤害、条件伤害、延迟触发、持续触发、补射、以及战后结算队列（反思/经验等）。
- 保持“先单敌人”前提不改 UI 敌人展示模型，但完善目标选择接口（为“所有敌人/随机敌人”等效果在单敌人下正确退化）。
- 将“最后一个弹匣”的语义改为动态：`last_wheel_index = wheels.size() - 1`，“只能放在最后一个弹匣”只需要在加入弹匣时进行检测，运行时不需要检测（如果玩家已经将该子弹插入第三个弹匣，然后拿到了第三个弹匣，该子弹可以正常工作。但是玩家不能将新的该类子弹插入第三个弹匣。）
- 完整实现 UI：
  - 战斗中弹匣逐条显示子弹，并对已消耗子弹做删除线标记。
  - “反思”提供可取消的删牌界面（仅作用于触发该卡的所在弹匣）。
  - “经验”提供额外一组“选牌奖励”（独立于常规战斗奖励）。
  - 战斗胜利后，按【补射】触发顺序依次处理“反思/经验/警惕”等结算，再进入常规战利品界面。

## Current State Analysis（基于仓库现状）

- 卡牌脚本接口：`static get_effect() -> Dictionary`，BattleController 目前只读取 `damage/block/move` 三字段结算（复杂效果无执行入口）。见 [battle\_controller.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/battle/battle_controller.gd#L422-L489)。
- 轮盘数据结构：RunContext/SlotMachine/BattleController 都是 `PackedStringArray` 的 id 列表；`roll()` 只返回技能 id 字符串，不含命中索引，无法定位“消耗的是哪一发”。见 [slot\_machine\_controller.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/battle/slot_machine_controller.gd#L27-L39)、[run\_context.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/systems/run_context.gd#L15-L39)。
- UI 的弹匣显示为调试汇总文本，不支持逐条子弹与删除线；奖励加入轮按钮固定 3 个（不支持未来 4/5 轮）。见 [ui\_controller.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/ui/ui_controller.gd#L581-L618) 与 [ui\_root.tscn](file:///e:/2025+/AI编程/漫游轮盘/scenes/ui/ui_root.tscn#L588-L682)。
- 战后流程：胜利后直接进入 RewardController 的常规三选一；未支持“战后额外流程/队列”。见 [game\_flow.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/core/game_flow.gd#L113-L131)、[reward\_controller.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/reward/reward_controller.gd#L21-L46)。
- 现有状态系统已能做“伤害修正/移动触发/回合开始”等 hook，但缺少统一的“敌人行动前后/回合结束”钩子，且没有临时力量、免疫下一击、复制下一次攻击等状态定义。见 [status\_container.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/systems/status_container.gd#L61-L84)。

## Assumptions & Decisions（已确认）

- 弹匣=轮盘（wheels），未来轮盘数量可能 > 3；“最后一个弹匣”按动态最后轮处理。
- 暂不做多敌人底座；“所有敌人/随机敌人”在单敌人下退化为主敌人。
- 临时力量在“下回合开始（玩家 on\_turn\_start）”清空。
- 需要全流程 UI：消耗删除线、反思删牌（可取消）、经验额外选牌，并且战后按【补射】触发顺序队列化处理后再进入常规战利品界面。

## Proposed Changes

### A) 轮盘/弹匣：引入“子弹条目（entry）”与可定位 roll 结果

**目标**

- 支持【消耗】（本战斗内移出可抽取池，但仍显示在弹匣并标记删除线）。
- 支持“从本弹匣/其它弹匣随机打出子弹”“从已消耗子弹池随机打出”等效果。

**改动点**

- [slot\_machine\_controller.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/battle/slot_machine_controller.gd)
  - 将 wheel 元素从 `String` 扩展为 `Dictionary` 条目：`{"id": String, "consumed": bool}`，并保持向后兼容：`set_wheels()` 仍接受旧 `PackedStringArray` 输入，内部转换成条目数组。
  - `roll()` 返回结构升级为 `Array[Dictionary]`（每轮一个）：`{"wheel_index": int, "entry_index": int, "id": String}`，并保证只从 `consumed=false` 的条目中抽取。
  - `get_pool_snapshot()` 返回深拷贝（便于 UI 渲染与战斗状态观测）。
- [battle\_controller.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/battle/battle_controller.gd)
  - `_slot_last_roll` 类型从 `Array[String]` 升级为 `Array[Dictionary]`（携带 wheel/index/id）。
  - `_emit_state()` 中 `state.slot.pools/last_roll` 同步升级结构。
  - `request_player_action("roll")` 的时间线循环改为按 `roll_items` 推进，并将 wheel/index 传入技能结算（用于消耗/随机打出定位）。

### B) 卡牌效果执行：从“三元 effect”升级为“动作序列（actions）+ 条件/触发/延迟”

**目标**

- 让 37 张枪手卡按描述生效（含：条件伤害、多段、随机、延迟、持续、补射等）。
- 仍保留旧卡兼容：若卡脚本只提供 `damage/block/move`，照旧结算。

**方案**

- 约定卡脚本 `get_effect()` 返回可选字段：
  - `actions: Array[Dictionary]`：顺序执行的动作列表（move/wander/deal\_damage/gain\_block/self\_damage/apply\_status/fire\_random\_bullet/repeat\_self/schedule 等）。
  - `barrage: Dictionary`：补射规则（`{"enabled": true, "when": "any_enemy_dead", "then_actions":[...]}`）。
  - `constraints: Dictionary`：构筑限制（如 `{"wheel":"last"}`）。
  - `meta: Dictionary`：战后效果（如 `{"post_victory_queue":[...], "order_key": ...}`），用于【经验/反思/警惕】等。
- 在 BattleController 新增一个执行器（可内嵌函数或独立脚本）：
  - 输入：`skill_id + roll_item(wheel_index/entry_index) + battle_ctx(turn/cell/block/... )`
  - 输出：step 结果（包含多段明细、是否触发补射、产生的战后队列条目等）。
- 关键上下文状态（BattleController 维护）：
  - `attacks_played_this_turn`（用于“第一张攻击”判定）
  - `forward_distance_this_turn`（用于滑铲等）
  - `pending_scheduled_events`（下回合触发）
  - `post_victory_queue`（战后队列，按触发顺序入队）

**文件改动**

- [battle\_controller.gd](file:///e:/2025+/AI编程/漫游轮盘/scripts/battle/battle_controller.gd)
  - 扩展 `_resolve_skill()` 支持 `actions` 执行与多段；并补齐 hook：`before_step/after_step`、`before_enemy_action/after_enemy_action`（用于示警等“一回合效果”可靠移除）。
  - 支持以下动作（覆盖 37 张卡需要）：
    - 位移：`move`（前进/后退）、`wander`（随机方向尝试位移，保持“即使边界也尝试”）、`teleport_to_cell`（实现为 delta 移动或直接设置）。
    - 伤害：`deal_damage`（支持多段 hits、目标 primary/all/random、按格子范围选敌）、`self_damage`（规则：直接扣 HP，不消耗格挡；写入计划作为一致性约束）。
    - 防御：`gain_block`。
    - 随机打出：`fire_random_bullet`（同弹匣/其它弹匣、available/consumed 过滤、次数、是否允许重复、是否排除当前条目）。
    - 延迟：`schedule`（在下回合开始执行一组 actions）。
    - 补射：当场上存在 `hp<=0` 的敌人时触发指定 actions（含“再发动一次该子弹”）。
    - 运行时资源：`gain_gold`（直接改 RunContext.gold）。
  - 【消耗】：当卡带 `tags` 含 “消耗”时，将命中条目标记 `consumed=true`；后续 roll/随机抽取默认排除 consumed。
- `scripts/cards/gunslinger_01.gd` \~ `gunslinger_37.gd`
  - 逐张补齐 `actions/barrage/constraints/meta` 的机器可读定义（不再依赖 notes 文本）。
  - 统一“攻击”判定：凡包含 `deal_damage` 动作的卡标记 `is_attack=true`（用于首攻与凝神复制判定）。

### C) 状态系统补齐：力量/临时力量/免疫下一击/复制下一次攻击/移动触发被动等

**目标**

- 用状态承载长期/一次性触发，减少 BattleController 硬编码复杂逻辑。

**新增/修改**

- [data/statuses.json](file:///e:/2025+/AI编程/漫游轮盘/data/statuses.json)：新增正式状态定义（可保留测试状态）：
  - `strength`：造成伤害 +X（player.before\_deal\_damage）
  - `temp_strength`：同上，但在玩家 `on_turn_start` 时清空
  - `next_attack_double`：下一次攻击发动两次（在 BattleController 处理“攻击卡”时消费）
  - `next_enemy_hit_zero`：下一次来自敌人的伤害变为 0（player.before\_take\_damage，触发后移除）
  - `move_forward_deal_damage`（死亡舞步）：每次前进触发对敌人伤害
  - `move_backward_gain_block`（绝望舞步）：每次后退触发获得格挡
  - `enemy_attack_halved_this_turn`（示警）：敌方本回合伤害减半（enemy.before\_deal\_damage），在 `after_enemy_action` 移除
  - `next_turn_gain_block`（掩蔽等延迟格挡）：player.on\_turn\_start 触发一次后移除
  - `next_time_take_damage_reflect`（自卫反射）：player.after\_damage(source=enemy且applied\_damage>0) 反伤并移除
- 对应脚本放到 `scripts/statuses/*.gd`，并在 BattleController 合适时机派发必要 hook（尤其 `after_enemy_action`）。

### D) 构筑限制：最后弹匣限制与动态轮盘数量

**目标**

**改动点**

### E) UI：消耗删除线 + 反思删牌界面 + 经验额外选牌

**目标**

- 战斗中显示每个弹匣的子弹列表（按顺序），已消耗用删除线。
- 反思：战后弹出删牌界面（仅所在弹匣），允许取消（不删直接退出）。
- 经验：战后弹出“额外一组子弹奖励”（逻辑上等价于再来一轮三选一加轮）。

**实现策略**

    - 维护一个 CardDatabase（或 CardScriptResolver）用于把 id 映射为名称。
    - `_render_slot_debug` 改为渲染条目列表：`consumed=true` 的条目用 `[s]名称[/s]`（或替代符号）标记。
- 反思删牌界面

  - 内容：列出触发卡所在 wheel 的所有子弹条目（显示名称），点击某项即从 RunContext 对应 wheel 永久移除该条目并关闭；“跳过”直接关闭不删除。

### F) 战后结算队列：按【补射】触发顺序执行，再进入常规战利品

**目标**

- 战斗结束后：
  1. 读取本场战斗累积的 `post_victory_queue`（由补射触发的卡入队，保序）
  2. 依次执行：反思（可取消）→经验（额外选牌）→……（未来扩展）
  3. 全部处理完后，再进入常规 `begin_battle_rewards()`

**改动点**

  - `FlowState` 扩展：增加 `POST_BATTLE_QUEUE` 与若干子状态（或用一个队列驱动器）。
  - `_on_battle_finished_victory(result)`：不再直接进入常规奖励；先保存队列并逐项驱动 UI 完成；队列清空后再调用 `_reward.begin_battle_rewards(_run)`。

## Verification Steps

1. 编译/运行检查

- Godot 脚本无诊断错误（GetDiagnostics 为空）。
- 战斗可正常开始、roll 时间线可正常跑完、胜利/失败流程不崩溃。

1. 关键机制用例（手动）

- 【消耗】卡：打出后同轮盘不再被 roll 到，但 UI 弹匣列表中仍存在并显示删除线。
- “随机打出子弹”：走火-松动扳机/枪口乱飘/上膛跌落/违规清理等能正确从指定弹匣（或已消耗池）选取并触发对应卡效果。
- “延迟触发”：走火-待击保管（下回合开始追加伤害）、掩蔽（下回合加格挡）生效。
- “持续触发”：死亡舞步（前进触发伤害）、绝望舞步（后退触发格挡）、英勇冲锋（前进叠临时力量，下回合开始清空）。
- “首攻判定”：精确射击/蓄势射击在本回合第一张攻击时强化；否则走常规数值。
- “凝神”：下一次攻击发动两次（且只消费一次该效果）。
- “示警”：敌人本回合伤害预览与实际结算都减半，且在敌人行动后移除。
- “自卫反射”：本回合下次实际受到敌人伤害时触发反伤并移除。
- “假目标”：下次敌人命中时伤害变 0，并切换玩家 token 立绘到 `设计方案/角色草稿（假目标）.png`，持续时间为本战斗内）。

1. 战后队列顺序（按用户给定示例）

- 若同一战斗中按顺序触发了反思→经验：
  - 胜利后先弹出反思删牌界面（可取消）；
  - 关闭后弹出经验额外选牌界面；
  - 关闭后进入常规战利品界面。
