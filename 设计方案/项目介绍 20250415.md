# 漫游轮盘dbg：设计稿v0.1 20250415

## 1. 游戏定位

这是一个轻度的 **类《幸运房东》构筑游戏**。

核心体验是：

> 玩家通过构筑老虎机随机池，摇出技能组合，看角色自动打出 combo。

主题是：

> 一个漫游枪手不断向右奔跑、战斗、成长，并通过随机技能组合击败敌人。

核心循环：

**进入战斗 → 敌人显示意图 → 玩家使用道具 → 摇老虎机 → 自动发动技能 → 敌人行动 → 战斗胜利 → 选择奖励 → 调整随机池 → 进入下一节点**

核心平台：
使用godot搭建，优先考虑手机端竖屏适配，最终导出一份html并部署在cloudfare pages上。
保留将项目导出为可执行文件的可能。

# 2. 战斗回合流程

每场战斗按固定流程循环。

| 阶段 | 内容                 |
| -- | ------------------ |
| 0  | 进入战斗               |
| 1  | 回合开始阶段             |
| 2  | 敌人生成本回合意图          |
| 3  | 自动结算暂停，玩家可以使用一次性道具 |
| 4  | 玩家摇老虎机             |
| 5  | 刷新出 3 个技能，角色按顺序使用  |
| 6  | 玩家回合结束阶段           |
| 7  | 敌人行动阶段，执行本回合意图     |

之后回到阶段 1。

整个流程中，只有两个阶段需要玩家手动操作：

- 使用一次性道具
- 摇老虎机

其他内容自动结算。

# 3.老虎机系统

## 3.1 基础结构

老虎机初始有 **3 个轮子**。

每个轮子对应一个独立随机池。

每回合摇老虎机时，每个轮子各随机出 1 个技能。

角色按顺序发动：

**轮子 1 → 轮子 2 → 轮子 3**

## 3.2 初始随机池

| 轮子  | 初始内容      |
| --- | --------- |
| 轮 1 | 3 打击，1 防御 |
| 轮 2 | 2 打击，2 防御 |
| 轮 3 | 1 打击，3 防御 |

## 3.3 技能加入

战斗胜利后，玩家可以从奖励中选择一个技能，并决定加入哪个轮子的随机池。

基础选项为：

- 加入轮 1
- 加入轮 2
- 加入轮 3
- 取消

如果后续获得更多轮子，加入选项也会增加。

***

# 4. 技能与 combo

## 4.1 技能执行顺序

默认顺序：

**轮 1 技能 → 轮 2 技能 → 轮 3 技能**

部分状态可以改变这个顺序。

例如：

| 状态 | 效果     |
| -- | ------ |
| 混乱 | 技能乱序发动 |

## 4.2 三连相同 combo

如果一次摇老虎机时，三个轮子刷出 **三个一样的技能**，一定触发 combo。

表现为：

> 再发动一次所有技能。

## 4.3 技能间 combo 示例

| 类型     | 效果                     |
| ------ | ---------------------- |
| 回音     | 本回合每发动过一次声音技能，此技能再发动一次 |
| 易伤     | 本回合敌人受到的伤害增加 50%       |
| 移动触发伤害 | 你本回合每移动过 1 格，发动一次伤害    |

***

# 5. 卡牌 / 技能数值模型

## 5.1 基础卡

| 卡牌 | 效果          |
| -- | ----------- |
| 攻击 | 右 1，造成 6 伤害 |
| 防御 | 左 1，获得 5 格挡 |

## 5.2 攻防倾向

| 卡牌类型  | 常见表现      |
| ----- | --------- |
| 进攻性卡牌 | 一般向右移动    |
| 防御性卡牌 | 一般向左移动或不动 |

## 5.3 后续卡牌模型

| 卡牌类型    | 期望数值        |
| ------- | ----------- |
| 不需要配合的卡 | 约 8 点数值     |
| 有配合的卡   | 约 12–16 点数值 |
| 高配合上限卡  | 可以更高        |

## 5.4 移动的定位

移动不占数值模型。

移动主要属于：

> 风味层 / 虚拟层

也就是说，强度模型主要看伤害、格挡、状态等实际数值，而不是把位移本身折算进去。

***

# 6. 格挡系统

格挡是：

> 回合开始时清空，用于阻挡攻击的防御手段。

设计倾向是：

> 鼓励玩家通过获得格挡保证存活，而不是主要依靠移动躲避攻击。

防御性饰品可以提高战斗稳定性。

示例：

| 饰品    | 效果                        |
| ----- | ------------------------- |
| 防御性饰品 | 第一个轮子中每有一个技能，回合开始时获得 1 格挡 |

***

# 7. 一次性道具

战斗奖励中，偶尔会出现类似《杀戮尖塔》药水的一次性道具。

特点：

- 一次性消耗
- 在摇老虎机之前使用
- 用于减少一部分随机性
- 使用窗口在回合流程第 3 阶段

***

# 8. 饰品系统

饰品是常驻增强效果。

来源包括：

- 精英战后
- Boss 战后
- 某些事件

## 8.1 饰品示例

| 饰品效果     | 说明                          |
| -------- | --------------------------- |
| 两个相同技能强化 | 每当 roll 到两个一样的技能，这个技能额外发动一次 |
| 拾取时移除技能  | 拾起此饰品时，可以选择 3 个老虎机中的技能移除    |

## 8.2 Boss 饰品池

Boss 战后的饰品有独立饰品池。

特点：

- 比其他来源的饰品更强
- 数值模型为：增加一个老虎机轮子，但承担一些小减益

***

# 9. 战斗场地

战斗舞台是一个横向 **1 × 7** 的场地，类似《将军对决》。

敌人通常出现在最右侧之外的 **第 8 格**。

敌人从右向左打出攻击判定。

## 9.1 多敌人情况

有时战场中会有 2–3 个敌人。

规则：

- 一定有一个敌人在第 8 格
- 其他敌人在 1 × 7 场地中游荡
- 场内敌人拥有碰撞体积
- 场内敌人造成碰撞伤害

***

# 10. 玩家移动与攻击

## 10.1 移动表现

角色整体上一直从左往右跑。

大部分技能也会推动角色向右移动。

战斗结束后，角色继续向右前进。

## 10.2 边界规则

| 情况        | 结果        |
| --------- | --------- |
| 最左侧尝试向左移动 | 普通战斗中移动取消 |
| 最右侧尝试向右移动 | 受到碰撞伤害    |

在精英和 Boss 战中：

| 情况        | 结果           |
| --------- | ------------ |
| 最左侧尝试向左移动 | 受到伤害，并刷新敌人意图 |

更新 2026-04-17：

当玩家尝试移动到格子外时，会尽可能移动（在格子2向左3，会移动到格子1）并发出信号告知敌人意图状态机玩家试图移动出界（需要告知是向左还是向右）。在大部分情境中，玩家只是会停在边缘的格子上。

在精英战与boss战场景中，玩家试图从左侧逃离可能会给精英上buff，玩家试图向右侧碰撞可能会导致玩家立刻受到碰撞伤害。具体的行为写在敌人脚本与敌人状态机中。

## 10.3 玩家攻击规则

玩家攻击：

- 无限距离
- 没有攻击判定区域
- 优先攻击右侧最近的敌人

***

# 11. 跳起状态

部分技能包含“跳起”动作。

跳起可以规避对地攻击判定。

但跳起状态只能维持 **1 招**。

例：

如果跳起技能出现在轮 2：

1. 轮 2 技能发动时跳起
2. 轮 3 技能发动时落地
3. 因此无法继续规避后续地面攻击

***

# 12. 敌人系统

## 12.1 敌人攻击

敌人的攻击有判定区域。

敌人的本回合攻击判定会在每回合开始时公示。

## 12.2 敌人意图状态机

敌人由意图状态机控制。

可用意图包括：

| 意图     | 内容         |
| ------ | ---------- |
| 攻击     | 对特定区域造成伤害  |
| 防御     | 获得防御或减伤    |
| 赋予状态效果 | 对玩家或自身施加状态 |
| 推拉玩家   | 改变玩家位置     |
| 移动     | 场内敌人可使用    |
| 获得力量   | 提高自身后续数值   |

## 12.3 敌人成长

敌人会有成长机制。

目的：

> 避免玩家只拿防御对抗随机性。

例如敌人可以通过“获得力量”提高后续攻击数值。

## 12.4 敌人与玩家相对位置

角色一般不会出现在敌人右侧，因为敌人有碰撞判定。

如果某个敌人发现角色在它右侧：

- 下回合意图变为“不攻击”
- 尽可能向右移动到角色右侧
- 如果第 8 格没有敌人，则移动到第 8 格

***

# 13. 战斗结束与场景推进

战斗胜利后：

1. 清除场上杂物
2. 卷动背景和角色，通常向左
3. 直到角色处于第 2 列
4. 展示战斗奖励
5. 玩家选择奖励
6. 背景继续向左卷动
7. 暗示主角继续向右移动
8. 下一个敌人刷新在最右侧

***

# 14. 奖励展示界面

战斗胜利后展示：

- 3 个技能图标
- 3 个技能名字

玩家选择其中一个后：

1. 技能图标移动到左侧
2. 右侧展示文字效果
3. 空地上出现假人演示该技能
4. 下方出现加入老虎机的选项

基础选项：

- 加入轮 1
- 加入轮 2
- 加入轮 3
- 取消

如果老虎机轮子更多，则加入选项更多。

***

# 15. 地图节奏

一局游戏分三层，其结构为：
**层1-层2-层3-最终boss**

每一层的结构为：
**层开始事件 → 小怪 → 小怪 → 小怪 → 休息处 → 小怪 → 小怪 → 饰品 → 精英 → 休息处 → 小怪 → 小怪 → 小怪 → 商店 → 休息处 → 层Boss**

其中：

> 小怪节点可能被随机替换为事件。

***

# 16. 商店与事件

除了战斗，还有事件和商店。

这些节点主要用于重新配置老虎机随机池。

可进行的操作包括：

- 花费金币移动技能
- 花费金币删除技能
- 调整随机池结构

***

# 17. 战损与恢复

由于敌人攻击有判定区域，且玩家可以移动，玩家平均战损不会太高。

但因为老虎机随机性存在，战损方差会很大。

表现为：

> 玩家可能突然被打一刀大的。

恢复规则：

- 战斗结束后不会自动回血
- 进入休息处时自动回满

***

# 18. 当前待定问题汇总

## 18.1 技能与 combo

- 三连相同触发的“再发动一次所有技能”是否会递归
- 回音类技能是否允许无限递归
- 易伤属于技能效果、敌人 debuff，还是本回合全局状态
- 混乱等状态如何影响三连 combo 的结算顺序

## 18.2 随机池

- 新技能是否有默认可加入轮子限制
- 不同轮子的奖励权重是否不同
- Boss 饰品增加的新轮子是否拥有独立随机池
- 第 4 轮及更多轮子的初始内容如何决定

## 18.3 战斗时序

- 玩家攻击与敌人行动之间是否存在反击或即时触发
- 跳起状态与敌人攻击结算的精确时点
- 敌人意图刷新是只在回合开始，还是也会被特定事件打断

## 18.4 数值

- 伤害、格挡、状态是否共用同一估值体系
- 敌人力量成长持续到战斗结束，还是有限回合
- 碰撞伤害的基础数值
- 最左侧惩罚伤害的数值
- 休息处是否只有回血功能

## 18.5 饰品

- 饰品数量是否有限制
- 普通饰品、精英饰品、Boss 饰品是否有稀有度分层
- Boss 饰品的小减益有哪些类型
- 防御性饰品在整体饰品池中的比例
